# P0 Global Qualifiers Rules

## Purpose

P0 Global Qualifiers is the first live content experiment of DC 2100 APEX OS.

It has two simultaneous functions.

Real-world function:

Identify which content variables create measurable audience response.

In-universe function:

Determine which vehicles, countries, drivers, conflicts, and audience tribes deserve future competitive arcs.

---

# 1. Duration

P0 runs for 7 consecutive days.

---

# 2. Publishing Volume

Daily target:

8–10 YouTube Shorts

Seven-day total target:

56–70 Shorts

Volume may be temporarily reduced only when:

- platform health requires intervention
- content quality collapses
- publishing functions fail
- policy or copyright risk is unresolved

---

# 3. Daily Mix Rule

Do not assign one country per day.

Every publishing day must mix:

- multiple countries
- multiple vehicle types
- multiple Hook structures
- multiple conflicts
- multiple story angles

The purpose is to reduce timing bias.

Example of forbidden structure:

Day 1 Japan only

Day 2 USA only

Day 3 Germany only

This makes country comparison unreliable.

---

# 4. Diversity Requirements

Country Diversity:

High

Vehicle Diversity:

High

Hook Diversity:

High

Conflict Diversity:

High

No single country, car, or format should dominate the full P0 test unless live performance data justifies reinforcement.

---

# 5. Content Mix Targets

News Reactive Content:

Approximately 20%

Experimental Content:

Approximately 10%

The remaining majority should test core automotive and DC 2100 narrative signals.

Percentages are operating targets, not rigid mathematical quotas.

---

# 6. Candidate Pool

Before each publishing day:

System prepares approximately 15–20 Candidates.

CEO selects approximately 8–10.

CEO actions:

- APPROVE
- REJECT
- REPLACE
- MORE SATIRE
- LESS SATIRE
- CHANGE CAR
- CHANGE COUNTRY
- SHORTER
- STRONGER HOOK
- REGENERATE

---

# 7. Candidate Evaluation

Each Candidate should be judged across:

- signal strength
- hook clarity
- vehicle appeal
- country relevance
- visual feasibility
- model feasibility
- satire potential
- DC 2100 fit
- originality
- long-term story potential

High virality alone is not sufficient.

A concept that cannot be produced reliably may be rejected.

---

# 8. Content ID

Format:

P0-{COUNTRY}-{CAR}-{NUMBER}

Example:

P0-JP-RX7-001

Rules:

- COUNTRY uses short uppercase country code.
- CAR uses a stable internal vehicle slug.
- NUMBER uses zero-padded sequence.
- Content ID must remain stable through the entire workflow.

Do not create a new Content ID for a retry or regenerated take.

---

# 9. Measurement Windows

Collect metric snapshots at:

1H

6H

24H

72H

7D

Early performance should not be judged from Views alone.

Where available, analyze:

- Views
- Likes
- Comments
- Subscribers Gained
- Watch Time
- Country Data
- Retention
- Engagement Rate
- View Velocity

---

# 10. Reinforcement Logic

The system should learn from combinations.

Examples:

Vehicle × Hook

Country × Conflict

Vehicle × Country

Hook × Story Role

Conflict × Satire Angle

Model × Shot Type

Do not conclude that one variable caused success when multiple variables changed simultaneously.

P0 is signal discovery.

Not perfect causal science.

---

# 11. Post-P0 Classification

Every tested content concept must eventually enter one of the following classes.

## P0 QUALIFIED

Strong enough to advance toward official Round 1 or future major competitive arcs.

## RESERVE SIGNAL

Shows meaningful secondary performance or fan demand.

Possible future uses:

- Underground Circuit
- Regional Qualifier
- Comeback Arc
- Retest

## REGIONAL ELIGIBLE

Shows meaningful potential in a specific country or region.

May return in future regional testing.

## ARCHIVED

Temporarily removed from active testing.

May return only when:

- new viral signal appears
- related country trend emerges
- related vehicle trend emerges
- story need changes
- audience requests return

---

# 12. Winner Reinforcement

A Winner is not copied blindly.

For successful content:

Identify:

- winning Hook
- winning visual structure
- winning vehicle signal
- winning country signal
- winning conflict
- winning satire intensity
- winning duration
- winning model

Then create controlled variations.

Do not create ten nearly identical copies of one hit.

---

# 13. Model Testing

During Pilot and early P0, track:

- Success Rate
- Retry Count
- Vehicle Consistency
- Motion Quality
- Time Cost
- Credit Cost

Before full P0 operation choose:

PRIMARY MODEL

SECONDARY MODEL

BACKUP MODEL

Do not distribute generation equally across every available model.

---

# 14. Channel Health Rules

Before P0 launch verify:

- channel operates normally
- uploads succeed
- analytics return correctly
- feature eligibility is normal
- copyright issues are unresolved only at zero count
- policy issues are unresolved only at zero count

Any serious unresolved platform issue can delay P0 launch.

---

# 15. YPP Mission

P0 contributes to the YPP growth mission.

Dashboard must track:

- Subscribers Current
- Subscribers Target
- 90-Day Shorts Views
- Expanded YPP Progress
- Full Monetization Progress
- Daily View Velocity
- Required Daily Velocity
- Gap to Target

Do not optimize exclusively for one viral video.

The objective is a repeatable growth engine.

---

# 16. P0 Production Rule

MVP production loop:

Creative Pack
→ CEO Approval
→ Copy Prompt
→ External AI Video Generation
→ Upload Result
→ QA
→ Publish Queue
→ YouTube
→ Analytics Return

Automatic video-generation APIs are outside P0 MVP scope.

---

# 17. P0 Safety Rule

P0 experimentation must not bypass:

- Canon
- platform safety
- copyright review
- political satire boundaries
- factual review for real-world news inputs

Fast publishing does not justify careless publishing.
